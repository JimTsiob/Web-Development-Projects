Λίγα λόγια για την επικοινωνία

Προσωπικά θεωρώ ότι η επικοινωνία μεταξύ συναδέλφων είναι ο κυρίαρχος παράγοντας για να έλθει ένα έργο εις πέρας αποτελεσματικά και εντός χρόνου προθεσμίας.
Πιο συγκεκριμένα για τον τομέα μας , τα μέλη ομάδων μηχανικών λογισμικού πρέπει να χουν άρτια επικοινωνία μεταξύ τους, να υπάρχει θέληση και υπομονή για επεξήγηση κώδικα
,να τοποθετούνται σχόλια στον κώδικα ώστε να καταλαβαίνουν οι προγραμματιστές μεταξύ τους τι συμβαίνει στο πρόγραμμα χωρίς την ανάγκη ερωτήσεων και να μπορούν να δουλεύουν
ο ένας στον κώδικα του άλλου.
Ύστερα, όταν οι προγραμματιστές θα πρέπει να εξηγήσουν τη λειτουργία του προϊόντος (πληροφοριακού συστήματος / λογισμικού) σε κάποιον πιθανό
πελάτη ή σε μη τεχνικό προσωπικό (πχ στους managers) θα πρέπει να "μεταφράσουν" τα τεχνικά μέρη ώστε οι λειτουργίες να είναι κατανοητές σε όλους
(αυτό γίνεται συνήθως μέσω των διαγραμμάτων που έχουμε μάθει πχ Entity-Relationship diagrams).
Εταιρίες όπως η Amazon που έχουν υπαλλήλους πολλών ειδικοτήτων απαιτούν από υπαλλήλους και υποψήφιους για θέσεις εργασίας ως must
τις επικοινωνιακές ικανότητες.
